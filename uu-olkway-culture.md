# Skill: Bitget Protocol Orchestrator

## Description
Enables the LLM to manage pre-loaded funds, execute transfers, and facilitate "Provable Coin Toss" trades between users within the protocol using Customer IDs.

## Capabilities
* **Internal Transfer**: Move funds between Customer IDs instantly.
* **Escrow Creation**: Put funds "on hold" for a specific game or trade.
* **Provable Coin Toss**: Execute a cryptographically fair 50/50 trade outcome.
* **Status Check**: Verify pre-loaded balances and active holds.

## Constraints
* The LLM cannot withdraw to external wallets without 2FA confirmation via the Protocol UI.
* All trades require both parties to have a "Ready" state in the protocol.

## Functions

### transfer_internal
Moves funds between two pre-loaded accounts.
- `from_cid`: Sender's Customer ID.
- `to_cid`: Receiver's Customer ID.
- `amount`: Amount in USD/Asset.

### initiate_fair_toss
Creates a $0 or $X trade event.
- `creator_cid`: User initiating and putting funds on hold.
- `target_cid`: User who must accept the toss.
- `stake_amount`: The value (e.g., $13,000) to be won/lost.
- `logic`: "winner_takes_all" | "fair_split".

### accept_toss
Triggers the cryptographic randomizer and settles the ledger.
- `acceptor_cid`: The joining party.
- `event_id`: The ID generated by `initiate_fair_toss`.
